# Deployment Guide

This guide covers repeatable local runs and basic container serving. It does not cover secured public hosting.

## Local Setup

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
```

Linux/macOS:

```bash
source .venv/bin/activate
```

Install dependencies and create artifacts:

```bash
pip install -r requirements-dev.txt
python -m src.data.generator
python -m src.models.train
python -m pytest
```

## Run The API

```bash
uvicorn src.models.api:app --host 0.0.0.0 --port 8000
```

Health check:

```bash
curl http://localhost:8000/
```

## Run Streamlit

```bash
streamlit run app.py
```

The app expects:

- `models/clinical_pipeline.joblib`
- `models/shap_background.joblib`

Run training first if either file is missing.

## Docker

```bash
docker build -t healthcare-ml-system .
docker run -p 8000:8000 -v ./models:/app/models healthcare-ml-system
```

## Docker Compose

```bash
docker compose up --build
```

Services:

- API: `http://localhost:8000`
- Streamlit: `http://localhost:8501`

Docker was not installed in the local validation environment, so the compose file still needs a build check on a machine with Docker.

## Environment Variables

See `.env.example`.

| Variable | Purpose |
| --- | --- |
| `MODEL_PATH` | Path to `clinical_pipeline.joblib` |
| `MLFLOW_TRACKING_URI` | MLflow backend URI |
| `ENABLE_MLFLOW_MODEL_LOGGING` | Set to `1` to log the model artifact to MLflow |
| `API_PORT` | API port used by hosting platforms |
| `STREAMLIT_SERVER_PORT` | Streamlit port |

## Hosted Options

Render or Railway can run the FastAPI container with:

```bash
uvicorn src.models.api:app --host 0.0.0.0 --port $PORT
```

Hugging Face Spaces can run the Streamlit app with:

```bash
streamlit run app.py --server.address 0.0.0.0 --server.port 7860
```

In both cases, the model artifact must be present before startup or generated by a release step.

## Operational Notes

- Do not load `joblib` files from user input.
- Add authentication before exposing the API publicly.
- Add request logging, model version metadata, and rollback steps for any hosted environment.
- Track drift and calibration if the data source changes.
